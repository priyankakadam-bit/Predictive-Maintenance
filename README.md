# Predictive Maintenance - Machine Failure Prediction

## Business Problem
Manufacturing downtime costs German industry billions annually.
This project uses Machine Learning to predict machine failures before they happen,
enabling proactive maintenance and reducing unplanned downtime by up to 40%.

## Dataset
- Source: AI4I 2020 Predictive Maintenance Dataset (UCI ML Repository / Kaggle)
- Size: 10,000 machine sensor readings
- Target: Binary classification - Machine Failure (0 = healthy, 1 = failure)
- Challenge: Only 3.4% failure rate, handled with SMOTE

## Methodology
1. EDA - visualize sensor distributions, check class imbalance, correlation heatmap
2. Data Cleaning - remove data leakage columns, encode categories
3. Feature Engineering - Power, Temp Difference, Wear Rate, Wear x Torque
4. SMOTE - fix class imbalance in training data
5. StandardScaler - normalize feature ranges
6. Model comparison: Logistic Regression vs Random Forest vs XGBoost
7. Streamlit app deployment

## Results

| Model               | F1 Score | ROC-AUC |
|---------------------|----------|---------|
| Logistic Regression | ~0.72    | ~0.84   |
| Random Forest       | ~0.83    | ~0.91   |
| XGBoost (best)      | ~0.87    | ~0.94   |

Note: In predictive maintenance, Recall is prioritized over Precision.
A missed failure (False Negative) costs far more than a false alarm.
